# Game Design Skill

## 核心循环设计（Core Loop Design）

**30秒测试原则**
每个游戏都需要一个有趣的30秒循环：
```
1. 行动 (ACTION) → 玩家做某事
2. 反馈 (FEEDBACK) → 游戏响应
3. 奖励 (REWARD) → 玩家感觉良好
4. 重复 (REPEAT)
```

## 游戏设计文档（GDD）

### 必备章节

| 章节 | 内容 |
|------|------|
| **创意概述** | 一句话描述 |
| **核心循环** | 30秒游戏玩法 |
| **机制** | 系统如何运作 |
| **进度** | 玩家如何进阶 |
| **美术风格** | 视觉方向 |
| **音效** | 声音方向 |

## 玩家心理学

### 动机类型

| 类型 | 驱动力 |
|------|--------|
| **成就者** | 目标、完成感 |
| **探索者** | 发现、秘密 |
| **社交者** | 互动、社区 |
| **杀手** | 竞争、支配 |

### 奖励机制

| 机制 | 效果 | 用途 |
|------|------|------|
| **固定** | 可预测 | 里程碑奖励 |
| **可变** | 成瘾性 | 战利品掉落 |
| **比例** | 基于努力 | 刷怪游戏 |

## 难度平衡

### 心流状态
```
太难 → 挫折 → 退出
太简单 → 无聊 → 退出
刚刚好 → 心流 → 参与
```

## 反模式（Anti-Patterns）

| ❌ 不要做 | ✅ 建议做 |
|----------|----------|
| 孤立设计 | 持续测试 |
| 在有乐趣之前打磨 | 先做原型 |
| 强制单一玩法 | 允许玩家表达 |
| 过度惩罚 | 奖励进度 |

> **记住：** 乐趣是通过迭代发现的，而不是在纸上设计的。
